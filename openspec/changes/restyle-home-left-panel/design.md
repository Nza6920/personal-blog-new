## Context

当前首页左侧区域由背景图、顶部两个按钮、简短句子和 welcome 文案构成，结构较单薄，无法承载参考图里的个人主页式信息分组。该页面仍基于 Blade 模板直接输出，左侧样式集中写在 `resources/views/home.blade.php` 内联样式中，并已包含主题切换与搜索开关的轻量 jQuery 交互。

这次改动目标集中在首页左侧视觉与信息架构升级，右侧文章流、搜索面板、分页与现有路由行为都需要保持稳定。实现上应尽量复用现有模板和前端资源组织方式，避免把一次视觉改版扩展成全站重构。

## Goals / Non-Goals

**Goals:**
- 将首页左侧改造成贴近参考图的沉浸式个人展示区，形成清晰的信息层级。
- 在左侧区域提供头像、主副标题、快捷入口、关于我、技术栈等模块化内容。
- 保留现有主题切换、首页入口、搜索和文章列表行为，确保首页功能不回退。
- 让左侧样式在桌面与移动端都能保持可读性与结构稳定。

**Non-Goals:**
- 不改动右侧文章列表的数据加载、分页或卡片排版逻辑。
- 不引入新的后端接口、数据库字段或 CMS 配置项。
- 不在本次改动中重做整站设计系统或抽离全局首页组件。

## Decisions

### 1. 在现有 `home.blade.php` 中直接重构左侧 DOM

左侧区域已经由 `#fh5co-aside` 承载，且首页的样式和脚本都集中在当前模板内。直接在这个模板里重构左侧 DOM，可以最小化影响范围，避免为单页改版引入额外组件层或跨文件耦合。

备选方案：
- 抽成新的 Blade partial：复用性会更好，但当前左侧区域和首页专属样式耦合较强，先在单文件内完成改版更稳妥。
- 改为 Livewire 组件：超出本次纯展示改版范围，也会抬高实现成本。

### 2. 继续使用背景图 + 覆盖层 + 毛玻璃卡片的视觉结构

参考图的核心风格是全高度背景、暗色蒙层、信息卡片和圆角按钮群。这与现有左侧背景图能力兼容，可以通过增强遮罩、内容容器、分组标题和 chip 样式来实现，而无需改动页面主骨架。

备选方案：
- 使用纯色侧栏：实现简单，但无法接近参考图的沉浸感。
- 把背景图替换为 CSS 渐变：视觉一致性较弱，且不符合用户给定参考方向。

### 3. 左侧文案与技术栈先采用模板内静态内容

当前首页左侧本就包含硬编码文案，参考图中的“关于我”和“技术栈”也更像展示性内容。先以模板内静态块实现，可以快速落地设计；若后续需要后台可配置，再单独提变更。

备选方案：
- 从数据库或配置文件读取：灵活性更高，但当前没有现成数据结构，超出本次范围。

### 4. 主题切换和入口按钮保留现有行为，仅更新外观与位置

现有主题切换逻辑已经基于本地存储工作正常。改版只需要调整按钮容器、尺寸、圆角、透明度和 hover 态，不应改动脚本语义，降低行为回归风险。

备选方案：
- 重写切换逻辑并加入更多主题态：与目标无关，风险更高。

### 5. 在移动端将左侧内容降为纵向内容区，而不是强行保持桌面覆盖层布局

参考图更接近移动视图，但当前站点还需要兼容桌面。实现时应让桌面保留沉浸式侧栏，移动端则通过调整间距、图标按钮换行和标签自适应来避免内容拥挤。

备选方案：
- 完全复刻参考图为单一窄屏布局：会破坏桌面首页的可用性。

## Risks / Trade-offs

- [左侧新增模块过多导致小屏拥挤] → 通过移动端断点调整内边距、分组间距和 tag 流式换行控制密度。
- [内联样式继续膨胀] → 将改动限制在首页左侧相关选择器内，避免顺手重排无关样式。
- [背景图与文案对比度不足] → 增加更深的遮罩和卡片背景，保证文字可读性。
- [硬编码内容后续需要频繁修改] → 本次先满足视觉改版，后续若需要配置化再单独演进。

## Migration Plan

该变更只影响首页模板和前端展示，无需数据迁移。发布时可直接替换模板；若需回滚，只需恢复 `resources/views/home.blade.php` 中左侧结构与相关样式。

## Open Questions

- 头像与背景图最终是否替换为新的素材文件，还是继续复用仓库现有资源，可在实现阶段根据现有图片质量决定。
