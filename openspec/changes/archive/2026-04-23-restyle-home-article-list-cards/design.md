## Context

首页左侧个人面板已经完成一次视觉升级，但右侧文章列表仍保留旧模板中的 `fh5co-entry` 结构：文章内容为单列文本，作者头像独立出现在卡片右侧。当前数据层虽然没有单独的首页封面字段，但文章详情页已经在使用 `Topic::background` 作为头图资源，因此这次调整需要在不改动数据模型和路由行为的前提下完成卡片重排，并尽量复用现有图片来源。

参考图展示的是深色横向文章卡片：左侧固定比例封面，右侧为文章基础文案区域。最终实现收敛为：删除右侧头像，左侧封面优先使用文章背景图，缺省时使用默认封面图；同时加入克制的卡片化视觉和暗色主题背景统一，但不扩展标签或底部附加信息层。

## Goals / Non-Goals

**Goals:**
- 将首页文章列表改为横向卡片布局，形成左侧封面、右侧正文的清晰结构。
- 在不新增字段的情况下，复用文章详情页现有的 `background` 资源作为首页封面来源，并在缺省时回退到默认图片。
- 保持首页文章链接、搜索面板、分页、主题切换和现有文案行为不变。
- 让新卡片在暗色主题和窄屏场景下仍保持稳定的间距、裁切和可读性。

**Non-Goals:**
- 不新增数据库字段、上传流程或文章封面管理能力。
- 不调整首页左侧个人面板的内容结构。
- 不在本次变更中引入新的交互动效、收藏按钮、文章统计来源改造或新的标签信息区。

## Decisions

### 1. 继续在 `resources/views/home.blade.php` 中完成结构与样式调整
首页左右布局和首页专属样式目前都内联在该 Blade 文件中，文章列表改版与左侧面板样式强耦合。继续沿用当前组织方式，能把改动限制在单一模板中，避免为一次视觉验证额外抽离组件或引入新样式入口。

备选方案：
- 抽离为独立 Blade partial 或组件。这个方向更利于复用，但当前只有首页使用该结构，抽离收益有限，会扩大本次改动面。

### 2. 复用文章背景图作为首页封面来源
文章详情页已经使用 `$topic->background` 作为头图来源，这比作者头像更符合“文章封面”的语义，也能保证首页和详情页的视觉资源一致。在背景图缺省时回退到站点默认封面图，可以避免空白封面，同时不需要新增任何后端字段或处理逻辑。

备选方案：
- 使用作者头像作为封面。这样能快速占位，但语义偏弱，而且与文章详情页的背景图来源不一致。

### 3. 仅保留当前文章基础信息，不预留新的附加信息层
新卡片只承载当前已存在的时间、标题和摘要，不在结构中提前加入标签、统计或其他占位区域。这样可以把改动聚焦在封面区引入和整体版式调整上，避免为了未来可能变化的内容过早设计额外容器。

备选方案：
- 预留参考图中的底部信息栏。这样更贴近效果图，但会把不存在的数据结构和视觉层带入当前版本，超出本次范围。

### 4. 使用纯 CSS 响应式适配，不新增脚本
改版目标集中在渲染结构和视觉样式，现有主题切换与搜索脚本无需感知文章卡片内部结构。响应式通过 media query 完成，减少行为风险。

备选方案：
- 使用 JS 动态调整卡片高度或裁切。没有必要，会增加复杂度并引入额外闪动。

## Risks / Trade-offs

- [文章背景图尺寸不一致导致封面裁切效果参差] → 通过固定封面比例和 `object-fit: cover` 降低视觉波动。
- [首页内联样式继续增长] → 将改动限制在文章列表相关选择器，并复用现有命名空间，避免无序扩散。
- [当前数据缺少参考图中的完整元信息] → 首版只保留现有可用信息，不伪造不存在的数据来源，也不添加空占位层。
- [深色主题下文字和卡片边界不够稳定] → 单独补充暗色主题下的页面背景、卡片背景、描边和 hover 色值，避免直接复用旧样式。
